# 数字花园博客 — 使用教程

## 访问方式

部署后在任意设备浏览器打开 `https://blog-7mr.pages.dev/` 即可访问。

---

## 四大模块

### 博客

- 首页 → 最近文章 → 点击进入阅读
- 文章详情页顶部进度条显示阅读进度
- 按 `←` `→` 切换上一篇/下一篇

### 日记

- 日期列表 → 点击日期进入日记
- 每篇日记用薄紫色卡片展示
- 心情和标签在顶部显示

### 待办

- **添加**: 输入文字 → 选优先级 → 可选截止日期 → 点"添加"
- **完成**: 点击圆形复选框勾选
- **编辑**: 双击或长按待办文字进入编辑模式，Enter 确认，Esc 取消
- **删除**: 点右侧垃圾桶图标，支持撤销
- **导出**: 点"导出"按钮 → 下载 JSON 文件备份
- **导入**: 点"导入"按钮 → 选择之前备份的 JSON 文件
- **快捷键**: 按 `N` 聚焦输入框，`Ctrl+Enter` 提交

### 日程

- **添加**: 输入标题 → 选日期/时间/时长 → 可选备注 → 点"添加"
- **查看**: 按日期分组，今天标记为绿色
- **删除**: 点右侧垃圾桶图标
- **导出/导入**: 同待办

## 跨设备同步

### 博客/日记（自动）

电脑本地写 Markdown → `git push` → Cloudflare 自动构建 → 所有设备可见

### 待办/日程（手动）

```
设备 A → 导出 JSON → 发送到设备 B → 设备 B 导入 JSON
```

常用传输方式：微信文件传输助手、AirDrop、邮件附件

## 本地写作指南

### 写一篇新博客

1. 在 `content/blog/` 下新建 `文章名.md`
2. 文件头部写 frontmatter：

```markdown
---
title: 文章标题
date: 2026-05-08
tags: [标签1, 标签2]
excerpt: 一段摘要
---

正文开始...
```

3. `git add -A && git commit -m "新文章" && git push`

### 写一篇新日记

1. 在 `content/diary/2026/` 下新建 `05-08.md`
2. 文件头部：

```markdown
---
date: 2026-05-08
mood: 😊
tags: [日常, 工作]
---

日记内容...
```

### 修改种子数据（待办/日程初始值）

编辑 `content/data/todos.json` 或 `content/data/schedule.json`，格式参考已有内容。

## PWA 安装

### iPhone (Safari)

1. 打开博客链接
2. 点底部中间的"分享"按钮
3. 往下滑 → 点"添加到主屏幕"
4. 自定义名称 → 点"添加"

### Android (Chrome)

1. 打开博客链接
2. 点右上角三个点
3. 点"添加到主屏幕"
4. 确认安装

安装后像普通 App 一样从桌面打开，支持离线访问已看过的页面。
