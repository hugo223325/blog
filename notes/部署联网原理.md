# 部署联网原理

## 整体流程

```
本地编辑 → Git Push → GitHub → Vercel 自动构建 → 全球 CDN 分发 → 任意设备访问
```

## 1. 内容是怎么变成网页的？

### 博客 / 日记（静态页面）

```
content/blog/*.md  →  Next.js 构建时读取  →  gray-matter 解析  →  remark 转 HTML
                     ↓
              generateStaticParams() 预生成所有页面
                     ↓
              纯 HTML 文件部署到 Vercel CDN 边缘节点
                     ↓
              用户访问 → CDN 直接返回 HTML（极快，无需服务器计算）
```

**关键点**：博客和日记在 `npm run build` 时就全部生成了静态 HTML。部署后只是把这些 HTML 文件分发到全球 CDN 节点，用户访问时不需要任何服务器运算。

### 待办 / 日程（客户端动态）

```
用户打开 /todo → 浏览器加载 React 组件 → 检查 localStorage
                                            ↓
                       有数据 → 直接渲染
                                            ↓
                       无数据 → fetch(/api/seed-data) → 加载种子 JSON → 渲染
                                            ↓
                       用户操作 → 即时写入 localStorage → 无需服务器
```

**关键点**：待办和日程的增删改操作完全在浏览器本地完成，不经过服务器。服务器只提供一个"种子数据"作为首次使用的初始值。

## 2. Vercel 部署原理

```
┌─────────────┐     git push      ┌──────────┐    自动触发     ┌──────────┐
│  本地代码    │  ───────────────→ │  GitHub   │ ────────────→ │  Vercel  │
└─────────────┘                   └──────────┘                └──────────┘
                                                                   │
                                                          npm run build
                                                             ↓
                                                       生成 .next/ 产物
                                                             ↓
                                                       分发到 CDN 边缘节点
                                                             ↓
                                          ┌─────────────────┼─────────────────┐
                                          ↓                 ↓                 ↓
                                     🇺🇸 美西节点        🇯🇵 东京节点       🇩🇪 欧洲节点
```

- **自动部署**：每次 `git push` 到 main 分支，Vercel 自动拉取代码并执行 `npm run build`
- **边缘网络**：构建产物分发到全球 CDN 节点，用户自动连接到最近的节点
- **HTTPS**：Vercel 自动提供 SSL 证书，无需手动配置
- **免费额度**：每月 100GB 带宽、100 次构建，个人使用足够

## 3. 为什么任何设备都能访问？

```
手机 ──→ 4G/5G/WiFi ──→ DNS 解析 xxx.vercel.app ──→ 最近 CDN 节点 ──→ 返回页面
电脑 ──→ 宽带 ──→ DNS 解析 xxx.vercel.app ──→ 最近 CDN 节点 ──→ 返回页面
平板 ──→ Wi-Fi ──→ DNS 解析 xxx.vercel.app ──→ 最近 CDN 节点 ──→ 返回页面
```

- **公网域名**：Vercel 提供 `*.vercel.app` 公网域名，任何联网设备都能访问
- **CDN 加速**：静态页面缓存在全球边缘节点，离用户越近越快
- **自适应布局**：Tailwind CSS 响应式设计，手机/平板/电脑都有良好体验
- **PWA 离线**：首次访问后 Service Worker 缓存页面，断网也能查看已访问过的内容

## 4. 数据是怎么跨设备同步的？

### 当前方案（手动）

```
设备 A                      Git 仓库                     设备 B
───────                    ────────                     ───────
写博客 → git push → ──→ content/blog/*.md ──→ ← git pull ← 博客更新
写日记 → git push → ──→ content/diary/*.md ──→ ← git pull ← 日记更新
待办 → 导出 JSON → 拖入 content/data/ → git push → Vercel 构建 → 设备 B 刷新页面（种子数据）
```

- 博客和日记通过 Git 天然同步，Vercel 自动部署
- 待办和日程需要手动导出 → 替换种子文件 → git push → 另一设备首次加载时获取

### 未来可升级方案

```
                          ┌─────────────┐
设备 A ───→ 读写 ───→    │  Vercel KV   │  ←─── 读写 ←─── 设备 B
                          │  (Redis)     │
                          └─────────────┘
```

- Vercel KV 是一个托管的 Redis 服务，免费额度 256MB
- 已预留 `StorageBackend` 接口，实现 `createVercelKVBackend()` 即可切换
- 切换后待办和日程实时跨设备同步

## 5. 离线访问原理（PWA）

```
首次访问
  │
  ├──→ 浏览器加载页面
  ├──→ Service Worker 注册并安装
  ├──→ 预缓存 HTML/CSS/JS 和静态页面（博客、日记）
  └──→ 缓存策略写入浏览器

后续访问（在线）
  │
  ├──→ Service Worker 拦截请求
  └──→ 缓存优先：有缓存直接返回，同时后台更新缓存

后续访问（离线）
  │
  ├──→ Service Worker 拦截请求
  └──→ 从缓存返回（博客/日记页面 + 静态资源）
  └──→ localStorage 数据始终可用（待办/日程）
```

**缓存策略**：
- 博客/日记页面：Cache First（优先用缓存，后台更新）
- 静态资源（CSS/JS）：Cache First
- API 路由：Network First（优先网络请求）
- 待办/日程数据：localStorage（无需网络）

## 6. 关键文件对照

| 文件 | 作用 |
|---|---|
| `next.config.mjs` | Next.js + Serwist PWA 配置 |
| `src/app/sw.ts` | Service Worker 源码 |
| `src/app/manifest.ts` | PWA 应用清单 |
| `src/lib/content.ts` | 读取 content/ 下的 Markdown 文件 |
| `src/lib/markdown.ts` | Markdown → HTML 转换 |
| `src/app/api/seed-data/route.ts` | 种子数据 API（Serverless Function） |
| `content/` | 所有内容文件 |

## 7. 常用操作

```bash
npm run dev          # 本地开发（localhost:3000）
npm run build        # 生产构建
npm run start        # 本地预览生产版本
git push origin main # 推送到 GitHub → Vercel 自动部署
```

## 8. 故障排查

| 问题 | 可能原因 | 解决 |
|---|---|---|
| 部署后页面空白 | 构建失败 | 查看 Vercel 构建日志 |
| 待办数据丢失 | 清除了浏览器数据 | 从 seed JSON 重新加载 |
| PWA 无法安装 | 缺少图标或 manifest | 确认 `public/icons/` 有图标 |
| 日记看不到 | 文件路径格式不对 | 检查 `content/diary/YYYY/MM-DD.md` |
