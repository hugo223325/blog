# PWA 配置记录

日期：2026-05-08
状态：已决定

## 背景

需要 PWA 支持以实现：
1. 移动端添加到主屏幕（类原生 App 体验）
2. 离线访问博客/日记内容
3. 待办/日程始终可用（localStorage）

## 决策

使用 **Serwist v9**（`@serwist/next` + `serwist`）：

- Service Worker 源码：`src/app/sw.ts`
- 编译输出：`public/sw.js`
- Manifest：`src/app/manifest.ts`（Next.js 自动生成）
- 构建集成：`next.config.mjs` 中 `withSerwistInit` 包装

## 技术细节

### 缓存策略
- `defaultCache` 来自 `@serwist/next/worker`
- 静态资源自动预缓存
- API 路由（seed-data）：运行时按需请求

### 图标
- 192x192 和 512x512 PNG
- 通过 `scripts/generate-icons.js` 生成纯色图标（#1a1a2e 深蓝色）
- 后续可替换为自定义图标

### 类型问题
- `ServiceWorkerGlobalScope` 在 Next.js 默认 TypeScript lib 中不可用
- 解决：使用 `WorkerGlobalScope & typeof globalThis` 并通过 global augmentation 扩展

## 验证方式

1. `npm run build` — 确认 sw.js 正确生成
2. 浏览器 DevTools → Application → Service Workers — 确认注册
3. 离线测试：断开网络 → 刷新 → 应显示缓存内容
