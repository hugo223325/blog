## 一、项目概述

个人全动态博客系统，支持博客、日记、待办、日程、体重五大模块。所有数据云端实时同步，任意设备打开即用。

### 技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| 前端框架 | Next.js 14.2.35 (App Router) | `output: "export"` 静态导出 |
| UI | React 18 + Tailwind CSS v3 | `darkMode: "class"` 手动切换 |
| 语言 | TypeScript 5 | 严格模式 |
| PWA | Serwist v9 (`@serwist/next`) | 离线缓存 + 安装到桌面 |
| 图标 | lucide-react 0.460 | SVG 图标库 |
| Markdown | remark + rehype + gray-matter | 浏览器端渲染 |
| API 层 | `_worker.js`（纯 JS，零依赖） | Cloudflare Pages 同域名部署 |
| 数据库 | Cloudflare D1 (SQLite) | 6 张表，主从架构 |
| 部署 | Cloudflare Pages + Wrangler | `wrangler pages deploy out/` |
| 认证 | 密码存 D1 + sessionStorage | `ensureAuth()` 统一验证 |
| 开发工具 | Node.js 22+ (wrangler), 18+ (Next.js) | nvm-windows 管理版本 |

## 三、数据库设计

### 数据库信息
- **类型**: Cloudflare D1 (SQLite 兼容)
- **名称**: `blog-db`
- **ID**: `def4da5a-a599-41d9-b700-a0690903d51d`
- **区域**: APAC

### 表结构

```sql
-- 用户设置（密码、身高等）
CREATE TABLE settings (
  key TEXT PRIMARY KEY,   -- 'password', 'height', 'goalWeight'
  value TEXT NOT NULL
);

-- 博客文章
CREATE TABLE posts (
  slug TEXT PRIMARY KEY,  -- URL 友好标识符
  title TEXT NOT NULL,
  content TEXT NOT NULL,   -- Markdown 原文
  excerpt TEXT DEFAULT '',
  tags TEXT DEFAULT '[]',  -- JSON 数组
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);

-- 日记
CREATE TABLE diary (
  date TEXT PRIMARY KEY,   -- '2026-05-28'
  content TEXT NOT NULL,
  mood TEXT DEFAULT '',
  tags TEXT DEFAULT '[]',
  updated_at TEXT NOT NULL
);

-- 待办
CREATE TABLE todos (
  id TEXT PRIMARY KEY,     -- UUID v4
  text TEXT NOT NULL,
  done INTEGER DEFAULT 0,  -- 0 或 1
  priority TEXT DEFAULT 'medium',  -- high | medium | low
  created_at TEXT NOT NULL,
  due_date TEXT             -- 可选截止日期
);

-- 日程
CREATE TABLE schedule (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  date TEXT NOT NULL,
  time TEXT DEFAULT '09:00',
  duration INTEGER DEFAULT 60,  -- 分钟
  notes TEXT DEFAULT '',
  created_at TEXT NOT NULL
);

-- 体重
CREATE TABLE weight (
  id TEXT PRIMARY KEY,
  date TEXT NOT NULL,
  weight REAL NOT NULL,    -- kg
  note TEXT DEFAULT '',
  created_at TEXT NOT NULL
);
```

### D1 注意点

1. **主从架构**: D1 使用 Primary-Replica 模式，写走主库，读可能走从库
2. **复制延迟**: 写入后立即 SELECT 可能拿到旧数据（已通过前端本地状态规避）
3. **本地开发**: `wrangler d1 execute` 默认本地，加 `--remote` 操作远程
4. **初始化**: `wrangler d1 execute blog-db --file=schema.sql --remote`

## 五、认证系统

### 架构

```
用户输入密码 → POST /api/auth → Worker 验证 D1 settings 表
  → 成功: sessionStorage.setItem("blog-password", pw)
  → 失败: 401 响应
```

### 认证流程

1. **首次访问**: sessionStorage 无密码
2. **写操作触发**: `ensureAuth()` 检测无密码 → 弹出 PasswordGate
3. **输入密码**: POST `/api/auth` 验证 → 成功存入 sessionStorage
4. **后续请求**: `request()` 自动读取 sessionStorage，加入 `Authorization: Bearer xxx`
5. **密码过期**: API 返回 401 → 自动清空 sessionStorage → 触发 PasswordGate 重登
6. **修改密码**: `/setup` 页面 → PUT `/api/settings/password`
7. **浏览器关闭**: sessionStorage 自动清除 → 下次打开需重新登录

### 关键实现

- `src/hooks/useAuth.tsx` — AuthProvider + `ensureAuth()`
- `src/components/ui/PasswordGate.tsx` — 密码输入弹窗
- `src/lib/api.ts` — `login()` / `request()` / 401 自动清密码
- `_worker.js` — `auth()` 函数 + `/api/auth` 路由

### 博客页面架构（重要）

```
page.tsx (10 行)
  └─ Suspense
      └─ BlogShell (逻辑层，~100 行)
          ├─ 状态管理: posts[], tags[], html, search
          ├─ URL 解析: useSearchParams → mode
          ├─ 数据加载: useEffect → api.getPosts/getTags
          ├─ CRUD 分发: handleCreate/Update/Delete → ensureAuth → API
          └─ 视图路由:
              ├─ ?new=1 → BlogCreate → Editor
              ├─ ?edit=1&slug=x → BlogEdit → Editor
              ├─ ?slug=x → BlogDetail
              └─ (default) → BlogList + 搜索 + 侧边栏
```

### 为什么用 `window.location.href` 而不是 `router.push`？

1. **Suspense 重挂载**: Next.js `<Suspense>` 在 `router.push` 同页导航时会卸载并重挂载子组件，导致所有 `useState` 重置
2. **全页重载保证数据新鲜**: 绕过一切缓存和状态问题
3. **模块级缓存保底**: `gPosts/gTags/gLoaded` 全局变量在页面重载期间保持（同一 session 内）

## 八、已知问题 & 解决方案

| 问题 | 原因 | 解决方案 |
|---|---|---|
| **PUT 后 GET 返回旧数据** | D1 主从复制延迟 | 前端用本地 state，不依赖 API 回读；编辑器保存后全页重载 |
| **JSX 编译报错 `Unexpected token div`** | Next.js 14 webpack bug：JSX 行过长（>200字符）导致解析失败 | 拆行，每个 prop/child 独立成行；或赋值给变量再返回 |
| **Service Worker 缓存 API 404** | Serwist 默认缓存策略缓存了 API 响应 | `/api/*` 路由使用 `NetworkOnly` 策略 |
| **中文标题导致 slug 为空** | `[^\w-]` 过滤掉所有中文字符 | `makeSlug()` 保留中文字符 + 时间戳后缀 |
| **CDN 缓存页面 HTML** | Cloudflare Pages 默认缓存静态文件 | 页面跳转时加 `?_t=时间戳` 参数 |
| **浏览器缓存 API 响应** | GET 请求默认可缓存 | Worker 所有响应加 `Cache-Control: no-cache, no-store` |
| **sessionStorage 密码过期** | 密码在 `/setup` 修改后，旧密码仍在 sessionStorage | `ensureAuth()` 先调 `/api/auth` 验证，失败则弹出重新输入 |
| **GitHub push 被墙** | HTTPS 443 阻断 | 用 `wrangler pages deploy out/` 直接上传到 Cloudflare |

## 十、深色模式实现

## 十二、Webpack Bug 规避指南

Next.js 14.2.35 存在 JSX 解析 bug，多行 JSX 在以下条件会触发：

- 函数体内有 `const` 声明后紧跟 `return (<JSX>)` — `(` 后换行
- JSX 属性行长度 > ~250 字符
- 多个 `const` 声明在 `const el = <JSX>` 之前

**规避方法（按优先级）**:
1. JSX 赋值给变量：`const el = <div>...</div>; return el;`
2. 所有属性独立成行，每行 < 150 字符
3. 函数体内只有一个 `const` 变量声明
4. 大组件拆分成多个小文件（每个 < 100 行）
